# AC测评工具 - 产品需求文档

## 1. 产品概述

**产品定位**：面向独立顾问的轻量级AC测评工具，采用"工具箱"模式，无需复杂项目管理，直接生成所需文档。

**核心价值**：将AC测评中最耗时的内容生成（胜任力模型、题本、报告）环节AI化，顾问聚焦专业判断。

**用户认证**：邮箱 + 密码登录

**技术架构**：React前端 + Python/Node.js后端API

**数据库**：SQLite（轻量、文件型）

**数据存储内容**：
- 胜任力模型
- 测评矩阵
- 题本内容
- 反馈内容
- 报告内容

### 2.1 生成端旅程（流A）

```
步骤1: 胜任力模型
├── 输入背景材料（可选，上传文件）
├── 输入能力名称（可选，多个用，分隔）
├── 输入能力数量（可选）
├── 点击"生成模型"
├── AI生成胜任力模型（能力名称、能力含义、行为标准）
├── 用户编辑确认（可增删修改所有文本）
└── 点击"提交定稿"（存储到数据库，后续覆盖）
    ↓ 自动跳转

步骤2: 评估矩阵
├── 查看已提交的胜任力模型
├── 选择要测评的能力（可一键全选）
├── 选择要使用的测评工具
├── 点击"生成矩阵"
├── AI生成评估矩阵
├── 直接提交 或 点击"编辑"
│   ├── 增删测评工具
│   ├── 增删能力
│   └── 调整能力与工具的配对
└── 点击"提交"（存储到数据库，后续覆盖）
    ↓ 自动跳转

步骤3: 题本生成
├── 展示5个题本卡片（名称、时长、状态、提交时间、下载、编辑入口）
├── 点击卡片进入题本编辑页
│   ├── 上传背景材料（可选，PDF/Word/Excel）
│   ├── 输入文字要求（可选）
│   ├── 点击"生成"
│   ├── 编辑生成的题本
│   └── 点击"提交"存入数据库
├── 可点击"下一个题本"或回到题本生成页面选择其他题本

步骤4: 评委手册
├── 点击"生成评委手册"
├── AI根据已提交的题本自动生成（无需用户上传材料）
├── 可编辑生成的评委手册
├── 点击"提交定稿"存入数据库
└── 下载Word/PDF
```

### 2.2 分析端旅程（流B）

```
步骤1: 上传测评记录
├── 下载固定模板（分数统计表 + 行为记录表）
├── 按模板填写被测者数据
├── 上传填写后的文件（Excel格式）
├── 系统解析并展示数据预览
├── 用户确认数据无误
└── 点击"保存记录"

步骤2: 生成反馈版报告
├── 选择被测者（单个或批量）
├── 点击"生成反馈版报告"
├── AI根据测评记录生成反馈信
│   ├── 一句话评价
│   ├── 优势项（能力名称、评价语、行为表现）
│   └── 不足项（能力名称、评价语、行为表现）
├── 可编辑报告内容
└── 点击"提交定稿"

步骤3: 生成组织版报告
├── 点击"生成组织版报告"
├── AI根据测评记录生成组织版报告（需先有反馈版）
├── 每份报告包含：个人评价、个人优势、个人不足、价值观/发展潜力、培养使用建议
├── 报告包含：各被测者评价、能力得分数据、总分数据、雷达图
├── 可编辑报告内容
└── 点击"提交定稿"

步骤4: 生成个人版报告
├── 点击"生成个人版报告"
├── AI根据测评记录生成个人版报告（需先有组织版）
├── 报告包含：开头段落、优势项、待提升项、发展建议、课程学习、书籍阅读
├── 报告包含：能力得分数据、总分数据、雷达图
├── 可编辑报告内容
└── 点击"提交定稿"
```

### 3.1 首页（Dashboard）

**功能**：工具箱入口，展示5大功能模块

**页面元素**：
- 顶部导航栏（Logo + 标题 + 用户头像/登录入口）
- 5个功能卡片（胜任力模型、评估矩阵、题本生成、上传记录、报告生成）
- 每个卡片：图标 + 模块名称 + 简短描述
- 底部：版本号 + 免责声明

**交互**：
- 点击卡片进入对应模块
- 未登录用户：卡片显示"本地模式"，登录后显示"已同步至云端"

## 4. 数据库设计

**数据库**：SQLite（轻量、零配置、文件型）

**数据策略**：每个用户只保留最新一份数据，后续数据覆盖之前的数据

### 4.1 用户认证表（users）

```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 4.2 胜任力模型表（competency_models）

每个用户只保留一条胜任力模型记录（覆盖更新）

**dimensions JSON结构**：
```json
[
  {
    "id": "uuid",
    "name": "创新突破",
    "meaning": "不墨守成规，用新方法、新技术、新思路解决问题，包括产品创新、技术创新、模式创新、管理创新等",
    "behavior_criteria": [
      {
        "id": "uuid",
        "title": "自我认知",
        "description": "对自身有清晰的认知，能认识到自己和团队的优势和短板，并对他人对自身的评估和建议保持坦诚开放的态度"
      },
      {
        "id": "uuid",
        "title": "拥抱变化",
        "description": "积极接纳新的事物和变化，对新事物/新观点保持开放和好奇，主动采用多种方法学习新知识"
      },
      {
        "id": "uuid",
        "title": "突破创新",
        "description": "突破原有框架，在工作中运用新思路、新知识、新方法、新技术、新工具有效解决实际问题"
      }
    ]
  }
]
```

```sql
CREATE TABLE competency_models (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER UNIQUE NOT NULL,
  name VARCHAR(255) NOT NULL,
  dimensions TEXT NOT NULL,  -- JSON: [{id, name, meaning, behavior_criteria: [{id, title, description}]}]
  source_files TEXT,          -- JSON: [file_id数组]
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 4.3 评估矩阵表（evaluation_matrices）

每个用户只保留一条评估矩阵记录（覆盖更新）

```sql
CREATE TABLE evaluation_matrices (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER UNIQUE NOT NULL,
  model_id INTEGER NOT NULL,
  tools TEXT NOT NULL,         -- JSON: [{id, name, weight, enabled}]
  matrix TEXT NOT NULL,         -- JSON: {dimension_id: {tool_id: boolean}}
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (model_id) REFERENCES competency_models(id)
);
```

### 4.4 测评记录表（assessment_records）

每个用户只保留一条测评记录（覆盖更新），存储上传的分数和行为数据

```sql
CREATE TABLE assessment_records (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER UNIQUE NOT NULL,
  model_id INTEGER,
  matrix_id INTEGER,
  upload_file_id INTEGER,
  score_table TEXT NOT NULL,      -- JSON: 分数统计表数据
  behavior_table TEXT NOT NULL,    -- JSON: 行为记录表数据
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (upload_file_id) REFERENCES files(id)
);
```

### 4.5 题本表（questionnaires）

每个用户保留5条题本记录（每种工具一条），覆盖更新

```sql
CREATE TABLE questionnaires (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  tool_id VARCHAR(50) NOT NULL,   -- beh/lgd/roleplay/case/vision
  model_id INTEGER NOT NULL,
  matrix_id INTEGER NOT NULL,
  content TEXT NOT NULL,           -- 题本内容
  word_url VARCHAR(500),
  pdf_url VARCHAR(500),
  status VARCHAR(20) DEFAULT 'completed',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (model_id) REFERENCES competency_models(id),
  FOREIGN KEY (matrix_id) REFERENCES evaluation_matrices(id),
  UNIQUE(user_id, tool_id)  -- 每个用户每种工具只有一条记录
);
```

### 4.6 评委手册表（judge_handbooks）

每个用户只保留一条评委手册记录（覆盖更新）

```sql
CREATE TABLE judge_handbooks (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER UNIQUE NOT NULL,
  model_id INTEGER NOT NULL,
  matrix_id INTEGER NOT NULL,
  questionnaires TEXT NOT NULL,   -- JSON: 关联的5个题本ID
  content TEXT NOT NULL,          -- 评委手册内容
  word_url VARCHAR(500),
  pdf_url VARCHAR(500),
  status VARCHAR(20) DEFAULT 'completed',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (model_id) REFERENCES competency_models(id),
  FOREIGN KEY (matrix_id) REFERENCES evaluation_matrices(id)
);
```

### 4.8 反馈材料表（feedbacks）

每个用户每个被测者只有一条反馈记录（覆盖更新）

```sql
CREATE TABLE feedbacks (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  record_id INTEGER NOT NULL,
  candidate_id VARCHAR(50) NOT NULL,
  candidate_name VARCHAR(100) NOT NULL,
  content TEXT NOT NULL,           -- 反馈信内容
  style VARCHAR(50),                -- formal/encouraging/professional
  word_url VARCHAR(500),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (record_id) REFERENCES assessment_records(id),
  UNIQUE(user_id, candidate_id)  -- 每个用户每个被测者只有一条记录
);
```

### 4.9 报告表（reports）

每种报告类型每个用户只保留一条记录（覆盖更新）

**报告类型**：
- `feedback_report`：反馈版报告（按被测者，每人一篇）
- `org_report`：组织版报告（每人一篇）
- `personal_report`：个人版报告（每人一篇）

```sql
CREATE TABLE reports (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  record_id INTEGER NOT NULL,
  report_type VARCHAR(50) NOT NULL,  -- feedback_report/org_report/personal_report
  candidate_id VARCHAR(50) NOT NULL, -- 被测者ID
  candidate_name VARCHAR(100),        -- 被测者姓名
  scores_data TEXT,                  -- JSON: 能力得分数据
  radar_chart_url VARCHAR(500),       -- 雷达图URL
  total_score DECIMAL(5,2),          -- 总分
  content TEXT NOT NULL,              -- 报告内容（JSON格式）
  language VARCHAR(20) DEFAULT 'zh',
  word_url VARCHAR(500),
  pdf_url VARCHAR(500),
  status VARCHAR(20) DEFAULT 'draft', -- draft/submitted
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (record_id) REFERENCES assessment_records(id),
  UNIQUE(user_id, report_type, candidate_id)  -- 每种报告每个被测者只有一条记录
);
```

**content JSON结构示例（反馈版）**：
```json
{
  "one_liner": "张三在沟通协作方面表现突出，问题解决有待提升...",

  ],

    { "competency": "问题解决", "comment": "目前存在...不足", "behavior": "..." }
  ]
}
```

**content JSON结构示例（个人版）**：
```json

  "opening": "XXX先生想法独立，内在坚定...",
  "strengths": { "competencies": ["用户思维", "目标管理"], "note": "最高分项、超过5.5分项" },
```

| 考生ID | 考生姓名 | 领导力 | 沟通协作 | 创新能力 | 分析思维 | ... |
|--------|----------|--------|----------|----------|----------|-----|

**行为记录表**：
| 考生ID | 考生姓名 | 测评环节 | 能力维度 | 行为描述 |
|--------|----------|----------|----------|----------|
| B1 | 王五 | BEI | 沟通协作 | 在描述项目经验时，能够清晰表达自己的角色和贡献... |
| B1 | 王五 | LGD | 领导力 | 在团队讨论中主动承担协调角色，推动达成共识... |

### 6.2 报告模板结构

#### 反馈版报告（第一步）
```
一句话评价：张三在沟通协作方面表现突出，问题解决有待提升...

优势项：
- 沟通协作：能力强，能够抓住协作方诉求，积极响应...；行为表现：能够抓住协作方诉求，面对难以化解的矛盾...
- 用户思维：...

不足项：
- 问题解决：目前存在...不足；行为表现：...
- ...
```

#### 组织版报告（第二步，每个被测者一份）
- 个人评价
- 个人优势
- 个人不足
- 价值观/发展潜力
- 培养使用建议（含使用建议、风险提示）

#### 个人版报告（第三步）
- 开头段落（先扬后抑式评价）
- 【优势项】（标注分数情况，如"最高分项、超过5.5分项"）
- 【待提升项】（标注分数情况，如"低于5分项"）
- 【发展建议】（3点编号列表）
- 【课程学习】（课程名、公司/平台、季度年份）
- 【书籍阅读】（书名列表）
