# BEI行为事件访谈模拟练习功能设计

## 1. 功能概述

BEI模拟练习功能允许用户与AI进行实时对话，模拟行为事件访谈过程。系统按照BEI题本中定义的胜任力逐个进行问答，直到收集到足够的行为事实依据后切换到下一胜任力。

### 核心特性

- 逐轮交互：AI提问 → 用户回答 → AI追问/判断终止
- 上下文隔离：每个胜任力的对话历史独立管理
- 实时语音输入：支持Web Speech API语音识别
- 进度可视化：显示当前胜任力进度和对话历史

### 3.2 会话状态

```python
class PracticeSession:
    id: int
    user_id: int
    tool_id: str = "beh"
    questionnaire_id: int

    competency_records: list[CompetencyRecord]
    created_at: datetime
    updated_at: datetime


class CompetencyRecord:
    id: int
    session_id: int
    competency_name: str
    competency_index: int
    
    messages: list[Message]  # 对话历史
    behavior_events: list[dict]  # 收集到的行为事件
    
    is_completed: bool = false
    completed_at: datetime


class Message:
    id: int
    record_id: int
    role: str  # "ai" | "user"
    content: str
    timestamp: datetime
```

### 4.1 REST API

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/practice/start` | POST | 开始练习，初始化会话 |

### 5.3 终止判断

AI判断是否已收集到足够的行为事实：

**终止条件**：
1. 至少收集到1个完整的行为事件（包含STAR四个要素）
2. 或者：AI已进行充分挖掘但用户无法提供更多具体例子

**终止判断Prompt**：
```python

{conversation_history}

# 判定标准

1. 已收集到至少1个具体的行为事件（包含：情境、任务、行动、结果）
2. 已进行多轮追问挖掘，但用户无法提供更多具体例子或回答模糊

请返回JSON格式：
```

## 7. 数据库设计

### 7.1 表结构

```sql
-- 练习会话表
CREATE TABLE practice_sessions (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    tool_id VARCHAR(50) NOT NULL,
    questionnaire_id INTEGER,
    questionnaire_content JSONB,
    current_competency_index INTEGER DEFAULT 0,
    status VARCHAR(20) DEFAULT 'in_progress',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 胜任力记录表
CREATE TABLE competency_records (
    id SERIAL PRIMARY KEY,
    session_id INTEGER REFERENCES practice_sessions(id),
    competency_name VARCHAR(100),
    competency_index INTEGER,
    messages JSONB DEFAULT '[]',
    behavior_events JSONB DEFAULT '[]',
    is_completed BOOLEAN DEFAULT FALSE,
    completed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 消息表（可选，独立存储）
CREATE TABLE practice_messages (
    id SERIAL PRIMARY KEY,
    record_id INTEGER REFERENCES competency_records(id),
    role VARCHAR(10) NOT NULL,
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 9. 安全性考虑

1. **认证**：所有API需通过JWT认证
2. **会话隔离**：用户只能访问自己的会话
3. **输入验证**：用户回答需要过滤敏感信息
4. **速率限制**：防止API滥用
